TATLICI YAŞAROĞLU — Yönetim paneli kurulumu
============================================

Menü (index.html) veriyi şu sırayla okur:
1) Netlify ortam değişkenleriyle üretilen js/config.generated.js (Supabase URL + anon key)
2) Supabase’deki menu_data tablosu
3) Yoksa data/menu.json ve js/menu-fallback.json

Yönetim: /admin.html — güçlü şifre kullanın, adresi müşterilere vermeyin.

QR: qr.html ve QR klasöründeki PNG, menü kök adresine kodlanır (şu an
https://peppy-sawine-0aba25.netlify.app/ ). Adres değişirse bu dosyaları güncelleyin.


--- A) Netlify (site + ortam değişkenleri) ---
1. Siteyi Netlify’a bağlayın (Git veya manuel deploy).
2. Site configuration → Build & deploy → Build settings:
   • Build command: npm run build
   • Publish directory: .
3. Site configuration → Environment variables → iki değişken (Scopes: Builds + Deploys):
   • SUPABASE_URL = Supabase Project Settings → API → Project URL
   • SUPABASE_ANON_KEY = anon / publishable public anahtar (service_role değil)
4. Değişkenlerden sonra yeniden deploy (gerekirse “Clear cache and deploy”).
5. Build log’da “npm run build” ve config üretimi görünmeli.


--- B) Supabase projesi ---
1. https://supabase.com → yeni proje.
2. SQL Editor’da supabase-schema.sql dosyasının tamamını çalıştırın.
3. Authentication → Providers → Email: açık olsun.
4. Authentication → Users → Add user ile yönetici oluşturun (e-posta onayı / Auto confirm ayarlarına dikkat).


--- C) Panele giriş ---
admin.html → oluşturduğunuz kullanıcı ile giriş → “Buluta kaydet” ile menüyü kaydedin.

İlk kayıttan önce menü boşsa: panelden bir kez kaydedin veya menu_data.payload’ı Table Editor’dan doldurun.


--- D) Supabase olmadan (yalnızca dosya) ---
1. admin.html → “Supabase olmadan yerel JSON düzenle”.
2. “menu.json indir” → projede data/menu.json ve js/menu-fallback.json içeriğini güncelleyin.
3. Siteyi yeniden yayınlayın.


--- Ürün görseli ---
• Supabase ile: “Dosya seç” (menu-images bucket) veya tam https:// Görsel URL.
• Supabase olmadan: yalnızca URL.


--- Güvenlik ---
• anon key tarayıcıda görünür; yazma RLS ile giriş yapmış kullanıcıya açıktır.
• service_role anahtarını asla siteye veya tarayıcıya koymayın.
