TATLICI YAŞAROĞLU — Yönetim paneli kurulumu
============================================

Menü sayfası (index.html) veriyi şu sırayla okur:
1) js/config.js içinde Supabase URL + anon key doluysa → Supabase’deki menu_data tablosu
2) Değilse → data/menu.json (statik dosya)

Yönetim adresi: /admin.html (Netlify’da herkese açık olur; güçlü şifre kullanın, linki müşterilere vermeyin.)


--- A) Sadece dosya ile (Supabase yok) ---
1. admin.html açın → "Supabase olmadan yerel JSON düzenle".
2. Değişiklikten sonra "menu.json indir" ile dosyayı indirin.
3. Projede data/menu.json dosyasının içeriğini bununla değiştirin.
4. Klasörü Netlify’a yeniden yükleyin.


--- B) Bulut ile (Supabase — önerilen) ---
1. https://supabase.com → yeni proje oluşturun.
2. Project Settings → API: Project URL ve anon public key’i not edin.
   Netlify’da ortam değişkeni olarak girin (NETLIFY-SUPABASE.txt).

3. SQL Editor’da supabase-schema.sql dosyasının tamamını çalıştırın.

4. Authentication → Providers → Email: açık olsun.
   Authentication → Users → "Add user" ile yönetici oluşturun.
   Önerilen e-posta ve şifre admin-giris-bilgileri.txt dosyasındadır.

5. admin.html → aynı e-posta/şifre ile giriş.
6. "Buluta kaydet" ile menüyü kaydedin. Ana sayfa birkaç saniye içinde güncellenir.

İlk kayıttan önce menü boş görünürse: yönetim panelinde bir kez kaydedin veya menu_data satırındaki payload’ı Table Editor’dan yapıştırın.


--- Ürün görseli ---
• Supabase ile: satırdaki "Dosya seç" ile yükleyin (menu-images bucket, herkese okunur).
• Veya "Görsel URL" alanına doğrudan https:// adresi yazın (ör. Imgur, kendi sunucunuz).
• Supabase olmadan: yalnızca URL yazılabilir; dosya yükleme çalışmaz.


--- Güvenlik ---
• anon key tarayıcıda görünür; yazma işlemi RLS ile yalnızca giriş yapmış kullanıcıya açıktır.
• service_role key’i asla js/config.js veya admin sayfasına koymayın.
