GitHub repo + Netlify — Admin paneli için sırayla yapın
========================================================

Bu klasörde git zaten hazır. Siz GitHub’da boş repo açıp aşağıdaki
komutlarla gönderirsiniz; Netlify bu repoyu bağlar. Supabase bilgisi
repoda değil, Netlify “Environment variables” içindedir.


1) GitHub’da yeni repo
----------------------
1. https://github.com → giriş → sağ üst “+” → New repository.
2. Repository name: örn. tatlici-yasaroglu-menu (boşluk yok, isteğe bağlı).
3. Public veya Private seçin.
4. ÖNEMLİ: “Add a README” / .gitignore / license EKLEMEYİN (boş repo).
5. “Create repository” deyin.
6. Sayfada çıkan HTTPS adresini kopyalayın, örn:
   https://github.com/KULLANICI_ADI/tatlici-yasaroglu-menu.git


2) Bilgisayardan ilk push (PowerShell)
--------------------------------------
Klasör: bu proje (Kadir Alyu)

  cd "C:\Users\Filthy\Desktop\Kadir Alyu"

  git add -A
  git status
  (admin-giris-bilgileri.txt ve js/config.secrets.js listede OLMAMALI)

  git commit -m "İlk yükleme: QR menü ve admin paneli"

  git remote add origin https://github.com/KULLANICI_ADI/REPO_ADI.git
  (Zaten remote varsa: git remote set-url origin ...)

  git branch -M main
  git push -u origin main

GitHub kullanıcı/şifre yerine genelde “Personal Access Token” ister;
GitHub → Settings → Developer settings → Personal access tokens.


3) Netlify
----------
1. https://app.netlify.com → Add new site → Import an existing project.
2. GitHub’ı seçin, yetki verin, az önce oluşturduğunuz repoyu seçin.
3. Build ayarı netlify.toml ile gelir:
   • Build command: npm run build
   • Publish directory: .
4. Environment variables (Deploy öncesi veya hemen sonra):
   • SUPABASE_URL = Project URL (https://....supabase.co)
   • SUPABASE_ANON_KEY = anon public key
   Scopes: Builds + Deploys — bkz. NETLIFY-SUPABASE.txt
5. Deploy site. Gerekirse “Clear cache and deploy site”.


4) Admin paneli adresi
----------------------
Yayınlandıktan sonra tarayıcıda:

  https://SITENIZ.netlify.app/admin.html

Supabase’de Auth kullanıcısı ve SQL şema: ADMIN-KURULUM.txt,
admin-giris-bilgileri.txt (yerelde, repoya gitmez).


5) Sonraki güncellemeler
-------------------------
Projede değişiklik yaptıktan sonra:

  git add -A
  git commit -m "Kısa açıklama"
  git push

Netlify otomatik yeniden derler ve yayınlar.
